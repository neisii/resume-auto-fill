## 6. 미매핑 라벨 현황

분류: **A** = aliases 추가로 해결 / **B** = profile 구조 신설 필요 / **–** = 자동화 불가

| 빈도 | 라벨 | 분류 | 비고 |
|---|---|---|---|
| 多 | `자격증`, `학력`, `교육` | B | 섹션 fill 지원 추가됨, alias 보완 여지 있음 |
| 多 | `사진`, `(사진)` | — | 이미지 셀, 입력 불필요 |
| 多 | `병역`, `성별` | B | profile 신설 가능 |
| 多 | `E-Mail`, `주소`, `연락처` | A | 현재 지원 |
| 多 | `yyyy.mm ~ yyyy.mm` | — | placeholder, 내장 패턴으로 자동 처리 |
| 多 | `희망연봉`, `보훈대상`, `추천인` | — | 자동화 불가 |
| 多 | `자기소개서`, 지원동기 | — | 자동화 불가 |

## 8. 자동화 한계 (구조적)

alias 확장이나 코드 수정으로 해결 가능한 문제와 그렇지 않은 문제를 구분한다.

**해결 가능**
- 새로운 라벨 표기 → aliases.yaml 추가
- 새로운 profile 필드 → profile + aliases 추가
- 예시값 채워진 셀 → `_overwrite` 추가

**구조적 한계 (해결 불가)**
- 회사마다 테이블 구조가 달라 자동화 가능 항목도 누락 발생
- 자기소개서, 지원동기, 희망연봉 등 맥락 의존 항목
- 단락형 이력서 (지원 형식 외)
- 이미지/도장/사진 셀

**실용적 운용 방식**  
이름·연락처·기술스택·프로젝트·경력처럼 매번 동일하게 반복되는 항목의 복붙 피로 제거가 핵심 가치다. 템플릿마다 100% 자동화를 목표로 하지 않는다. 나머지는 사람이 채우는 구조가 현실적이다.
